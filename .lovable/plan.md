

## AI-чат-бот для помощи покупателям

### Что будет сделано

1. **Кнопка чата** — плавающая кнопка в правом нижнем углу на всех страницах магазина (не в админке)
2. **Форма входа в чат** — имя, телефон, чекбокс «Принимаю политику конфиденциальности»
3. **Чат с ИИ** — бот знает каталог товаров, помогает выбрать букет, считает стоимость, оформляет заказ
4. **Эскалация** — кнопка «Позвать оператора» создаёт тикет
5. **Админка: раздел «Чаты»** — все диалоги видны админу, тикеты выделены

---

### База данных (2 новые таблицы)

**chat_sessions** — сессии чата:
- `id`, `customer_name`, `phone`, `status` (active / escalated / closed), `created_at`, `updated_at`

**chat_messages** — сообщения:
- `id`, `session_id` (FK → chat_sessions), `role` (user / assistant / system), `content`, `created_at`

RLS: вставка доступна всем (анонимные пользователи), чтение/обновление — только админам. Для сессии — пользователь может читать/писать по session_id, переданному клиентом.

### Edge Function: `chat-bot`

- Принимает `session_id` + `message`
- Загружает каталог товаров из БД
- Загружает историю сообщений сессии
- Формирует системный промпт с каталогом и инструкциями (помочь выбрать, посчитать, оформить заказ)
- Стримит ответ через Lovable AI (модель `google/gemini-3-flash-preview`)
- Сохраняет сообщения пользователя и ассистента в `chat_messages`
- При оформлении заказа — вызывает tool call для создания записи в `orders`

### Технические детали

- Стриминг SSE через edge function для плавного вывода ответов
- Системный промпт включает полный каталог товаров (название, цена, категория, наличие)
- Бот умеет: рекомендовать букеты, считать стоимость, собирать заказ, оформлять заявку в БД
- При эскалации — статус сессии меняется на `escalated`, видно в админке
- `react-markdown` для рендера ответов бота
